## üóÑÔ∏è Database Schema

### New Tables

**1. `email_management`**
```sql
- id (uuid, primary key)
- user_id (uuid, foreign key to auth.users)
- company_name (text)
- company_email (text)
- position (text)
- keywords (text[])
- subject (text)
- email_body (text)
- status (enum: new, under_review, approved, rejected)
- created_at (timestamp)
- updated_at (timestamp)
```

**2. `chatbot_sessions`**
```sql
- id (uuid, primary key)
- user_id (uuid, foreign key to auth.users)
- email_id (uuid, foreign key to email_management)
- conversation_history (jsonb)
- created_at (timestamp)
- updated_at (timestamp)
```

**3. `activity_logs`**
```sql
- id (uuid, primary key)
- user_id (uuid, foreign key to auth.users)
- activity_type (text)
- description (text)
- level (enum: info, warning, error, success)
- metadata (jsonb)
- created_at (timestamp)
```

### Security

All tables have Row Level Security (RLS) enabled with policies ensuring users can only access their own data.

## üîß Setup & Configuration

### Prerequisites
- Node.js 18+
- Python 3.12+
- Supabase account
- OpenAI API key

### Quick Setup

```bash
# 1. Clone repository
git clone https://github.com/ak-1344/AgentM.git
cd AgentM

# 2. Setup backend
cd backend
python3 -m venv /tmp/agentm-venv
source /tmp/agentm-venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# Edit .env with your credentials

# 3. Setup frontend
cd ../frontend
npm install
cp .env.example .env.local
# Edit .env.local with Supabase credentials

# 4. Setup database
# Run SQL scripts in Supabase SQL Editor:
# - database/schema_phase1.sql
# - database/schema_email_management.sql

# 5. Start servers
# Terminal 1 - Backend
./scripts/backend.sh start

# Terminal 2 - Frontend
cd frontend && npm run dev

# 6. Access application
# Frontend: http://localhost:3000
# Backend: http://localhost:8000
# API Docs: http://localhost:8000/docs
```

## üîÑ Migration Guide

If you were using a pre-release version:

1. **Database Migration**
   ```sql
   -- Run new schemas
   \i database/schema_email_management.sql
   ```

2. **Update Environment Variables**
   - Add `SECRET_KEY` (generate with `openssl rand -hex 32`)
   - Add `ENCRYPTION_KEY` (generate with Python Fernet.generate_key())
   - Update `CORS_ORIGINS` to JSON array format

3. **Update Dependencies**
   ```bash
   # Backend
   pip install -r requirements.txt
   
   # Frontend
   npm install
   ```

4. **Restart Services**
   ```bash
   ./scripts/backend.sh restart
   cd frontend && npm run dev
   ```
